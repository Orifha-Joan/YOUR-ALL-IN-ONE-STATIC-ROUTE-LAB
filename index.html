Scenario: All-in-One Static Routing Lab

This lab provides students with a hands-on experience configuring static routing in a simplified but practical network topology. The goal is to help learners understand how traffic is manually directed between networks without relying on dynamic routing protocols.

In this topology, students will configure and test three types of static routes:

Standard Static Route

A basic route that manually points to a specific next-hop IP address or exit interface to reach a remote network.

Default Static Route

A route that directs all traffic destined for unknown networks (0.0.0.0/0) to a specified next-hop. It acts as a "catch-all" route, especially useful for edge routers.

Floating Static Route

A backup route that has a higher administrative distance than the primary route. It only takes over when the primary route fails, providing basic redundancy.

Standard Static Route configuration

Currently only device that are direcly connected can communicate over the network, we will be a enabling a horizontal way communication


we will be doing this by explicitly specifying the Next-Hop to get to a network.

On router-LAG-1 lets do a standard static route to enable communication between servers 

router-LAG-2 > enable
router-LAG-2 # config terminal 
router-LAG-2 (config) # ip route 10.10.11.0 255.255.255.128 10.10.10.2


router-LAG-1 > enable
router-LAG-1 # config terminal 
router-LAG-1 (config) # ip route 172.16.11.0 255.255.255.128 10.10.10.1

The above command tell each router how to handle traffic coming from the repeated networks, so for Router-LAG-1 where the Netzone server is located and under the network of 10.10.11.0, the command say from Router-LAG-1 when it get traffic going to the Netclan server which is under the networkof 172.16.11.0 it should use the interface of router-LAG-2 10.10.10.1 which is the Next-hop of the directly connected interface and a vice verse on Router-LAG-2

Repeat the step for Router-ABJ-1 and Router-ABJ-2 using the appropriate Next-Hop

DEFAULT STATIC ROUTE


For to default static route we will simulating a External address.


On Router-ABJ-2 configure a loopback address. 

  Router-ABJ-1 > enable
  Router-ABJ-1 # config terminal 
  Router-ABJ-1 (config) # interface loopback0
  Router-ABJ-1 (config) # ip address 1.1.1.1 255.255.255.255
  

  Router-ABJ-2 > enable
  Router-ABJ-2 # config terminal 
  Router-ABJ-2 (config) # interface loopback0
  Router-ABJ-2 (config) # ip address 8.8.8.8 255.255.255.255

For the default route we are pointing to no Network or Subnet

so we will configure default route on router. 

  Router-ABJ-1 > enable
  Router-ABJ-1 # config terminal 
  Router-ABJ-1 (config) # ip route 0.0.0.0 0.0.0.0 203.40.133.2
  

  Router-ABJ-2 > enable
  Router-ABJ-2 # config terminal 
  Router-ABJ-2 (config) # ip route 0.0.0.0 0.0.0.0 203.41.133.1
